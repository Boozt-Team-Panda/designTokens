1) Overview

Component name: Button v2
Purpose: Enhanced button component with comprehensive variant support and improved accessibility.
Common aliases: Enhanced Button, Advanced Button, M3 Button, Comprehensive Button

‚úÖ Do

  • Keep labels concise (1-3 words), use action verbs

‚ùå Don't

  • Avoid for secondary information display, decorative elements, multiple primary actions on same screen
  • Don't truncate button text

4) Layout Rules

  • Container sizing: {Size.ContainerHeight.s|m|l} as minHeight tokens (s=32px, m=48px, l=48px).
  • 48 px min touch target: All interactive components must provide at least 48 px in one dimension. If the visual design is smaller, add outer/invisible padding.
  • Padding: Horizontal/vertical padding from {Spacing.Container.Horizontal.*} (8px s, 16px m, 20px l).
  • Element gaps: {Spacing.Element.Gap} between leading / content / trailing (8px).
  • Alignment logic: Center aligns content by default, start/end distribute children accordingly.
  • Focus/ink: Outer wrapper draws focus/elevation; inner surface draws bg + radius (8px corner radius).

7) Accessibility

  • Hit target: min 48 px (M3 baseline) across all platforms.
  • Keyboard & screen reader: roles, aria/traits, focus order.
  • Scaling: size tokens are minHeight; components expand if text scales (Dynamic Type, font scaling).
  • Contrast: meet WCAG AA/AAA as specified.
  • Assistive text: for inputs, error/help text linked via aria-describedby.

8) Platform Adapters

Web: CSS vars, min-height: 48px, flex/grid alignment, outer focus ring.
iOS: SwiftUI/UIKit, .frame(minHeight: 48), Dynamic Type, avoid clipping focus.
Android: Compose Modifier.heightIn(min = 48.dp), ripple on shaped surface, respect FontScale.

9) Events & Data

  • Events: onPress/onClick, onChange, onFocus/onBlur, etc. (with payloads).
  • Controlled vs uncontrolled: for inputs/toggles, specify source of truth.

10) Content Rules

  • Text: casing, max length, truncation vs wrapping.
  • Icons/media: permitted sizes, placement rules, empty states.

11) Performance & Theming

  • No layout shift: keep dimensions stable across states.
  • Theme-ready: all visuals via semantic tokens; no hard-coded colors.
